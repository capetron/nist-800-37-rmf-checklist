# NIST SP 800-37 Rev. 2: RMF Compliance Checklist

Use this checklist to track your organization's progress through the seven steps of the Risk Management Framework. Each item maps to specific tasks defined in NIST SP 800-37 Rev. 2. Mark items as complete when evidence is documented and approved.

**Source:** [NIST SP 800-37 Rev. 2](https://csrc.nist.gov/publications/detail/sp/800-37/rev-2/final)

---

## Step 1: Prepare (Organization Level)

- [ ] Assign Risk Executive (Senior Accountable Official for Risk)
- [ ] Assign Chief Information Officer (CIO)
- [ ] Assign Senior Agency Information Security Officer (SAISO/CISO)
- [ ] Establish risk management strategy (reference: NIST SP 800-39)
- [ ] Conduct organization-wide risk assessment (reference: NIST SP 800-30)
- [ ] Identify, document, and publish organizationally common controls
- [ ] Establish continuous monitoring strategy at the organization level (reference: NIST SP 800-137)
- [ ] Define and document mission and business process priorities
- [ ] Establish organizational governance structure for risk management
- [ ] Approve and disseminate organization-wide security and privacy policies

## Step 1: Prepare (System Level)

- [ ] Identify and register the system in the organizational inventory
- [ ] Assign System Owner
- [ ] Assign Information System Security Officer (ISSO)
- [ ] Identify the Authorizing Official (AO) for the system
- [ ] Define the authorization boundary
- [ ] Identify information types processed, stored, and transmitted
- [ ] Identify system stakeholders
- [ ] Identify assets requiring protection
- [ ] Conduct system-level risk assessment
- [ ] Define the system development life cycle (SDLC) phase
- [ ] Identify enterprise architecture integration points

## Step 2: Categorize

- [ ] Document all information types per NIST SP 800-60 Vol. 1 and Vol. 2
- [ ] Assign confidentiality impact level (Low, Moderate, High) for each information type
- [ ] Assign integrity impact level (Low, Moderate, High) for each information type
- [ ] Assign availability impact level (Low, Moderate, High) for each information type
- [ ] Apply high-water mark to determine overall system categorization (FIPS 199)
- [ ] Document categorization rationale and any adjustments
- [ ] Obtain Authorizing Official review and approval of categorization
- [ ] Update system registration with approved categorization
- [ ] Record categorization in the System Security Plan (SSP)

## Step 3: Select

- [ ] Identify the initial control baseline from NIST SP 800-53B (Low, Moderate, or High)
- [ ] Review and apply scoping considerations to baseline controls
- [ ] Identify and document compensating controls where baseline controls are not feasible
- [ ] Define organization-defined parameters for applicable controls
- [ ] Allocate controls as system-specific, hybrid, or common
- [ ] Document inherited common controls and identify common control providers
- [ ] Obtain common control provider documentation and assessment results
- [ ] Identify any additional controls beyond the baseline based on risk assessment
- [ ] Document all selected controls in the SSP with implementation responsibility
- [ ] Develop or update the system-level continuous monitoring strategy
- [ ] Review and approve the SSP (System Owner and AO concurrence)

## Step 4: Implement

- [ ] Implement all system-specific security controls per SSP
- [ ] Implement all system-specific privacy controls per SSP
- [ ] Implement hybrid control portions assigned to the system
- [ ] Configure system settings per organizational security configuration standards
- [ ] Document implementation details for each control in the SSP
- [ ] Record implementation methods, mechanisms, and technologies used
- [ ] Identify responsible parties for each implemented control
- [ ] Document any deviations from planned implementation in the POA&M
- [ ] Update the SSP with actual (as-built) implementation details
- [ ] Verify implementation aligns with enterprise architecture requirements

## Step 5: Assess

- [ ] Develop the Security Assessment Plan (SAP)
- [ ] Define assessment scope (all controls for initial, subset for ongoing)
- [ ] Define assessment procedures per NIST SP 800-53A (examine, interview, test)
- [ ] Select or engage qualified Security Control Assessor(s)
- [ ] Ensure assessor independence requirements are met
- [ ] Conduct security control assessment per the SAP
- [ ] Conduct privacy control assessment per the SAP
- [ ] Document all findings in the Security Assessment Report (SAR)
- [ ] Categorize findings by severity (Critical, High, Moderate, Low)
- [ ] Identify operational remediation actions for each finding
- [ ] Create or update the Plan of Action and Milestones (POA&M)
- [ ] Assign POA&M item owners and remediation target dates
- [ ] Provide SAR and updated POA&M to the Authorizing Official

## Step 6: Authorize

- [ ] Assemble the complete authorization package (SSP, SAR, POA&M)
- [ ] Include supporting documentation (risk assessment, interconnection agreements, etc.)
- [ ] Submit authorization package to the Authorizing Official
- [ ] AO (or designated representative) reviews the authorization package
- [ ] AO conducts or reviews risk determination based on SAR findings and residual risk
- [ ] AO considers organizational risk tolerance and mission requirements
- [ ] AO issues authorization decision: ATO, DATO, or IATT
- [ ] Document authorization decision with terms, conditions, and expiration (if applicable)
- [ ] Communicate authorization decision to system owner, ISSO, and stakeholders
- [ ] Update system inventory with authorization status and date
- [ ] Archive authorization package per records management requirements

## Step 7: Monitor

- [ ] Execute the continuous monitoring strategy
- [ ] Monitor configuration changes to the system and environment
- [ ] Conduct ongoing security and privacy control assessments per schedule
- [ ] Perform vulnerability scanning at defined intervals
- [ ] Analyze monitoring results and security alerts
- [ ] Report security and privacy posture to AO at defined intervals
- [ ] Review and update the SSP to reflect current system state
- [ ] Review and update the SAR with ongoing assessment results
- [ ] Review and update the POA&M (close resolved items, add new findings)
- [ ] Evaluate the impact of proposed and actual changes on authorization
- [ ] Conduct annual security reviews (at minimum)
- [ ] Determine if changes require re-authorization
- [ ] Maintain ATO through ongoing authorization activities
- [ ] Report significant security events to the AO and SAISO
